# AI-SYSTEM-MAP — Pygmalion / C.R.I.S.T.A.L.L.

> Единая карта системы для AI-агентов (OpenCode, Claude, Grok, ChatGPT, Jules).
> Снижает drift между сессиями и предотвращает «галлюцинации оптимизации».

---

## Mission

Пространство «Пигмалион / К.Р.И.С.Т.А.Л.Л.» — институт осознанного выбора и мышления в числовой НОД.- Платформе "П.\К." 7-го технологического уклада. Цель — создание условий для добровольного, осознанного присутствия человека без принуждения, монетизации и геймификации.

## 2. Canon Layer (SSOT)

**Путь:** `backend/core/canon/`
**Версия:** `phase1-stable-2026.05`
**Импорт:** `const Canon = require('./core/canon');`

### Модули

| Файл | Статус | Описание |
|---|---|---|
| `emission-policy.js` | ✅ Confirmed | SSOT лимитов эмиссии: min 3, max 13, триады T1-T5, silence 19:55–20:00, sleep 20:00–04:00 |
| `grammar.js` | ✅ Confirmed | Валидация О.К. (Grammar Engine: `::key::`) |
| `ontology.js` | ✅ Confirmed | Фазы (silence/sleep/active/impulse), статусы У.Е., типы актов, триады |
| `temporal.js` | ✅ Confirmed | 24+4 правило, burn_at, getPhase(), canAct() |
| `reserved.js` | ✅ Confirmed | Системный резерв ::00::–::33::, ОРАКУЛ-С `::О::`, префиксы |
| `bridges.js` | ✅ Confirmed | Bridge-symbols — связывание алфавитов |
| `protocols.js` | ✅ Confirmed | Протоколы ::про.1::–::про.12::, проверка доступа |
| `index.js` | ✅ Confirmed | SSOT entry point, экспортирует все модули |

### Правила работы с Canon

1. **Любой код импортирует ТОЛЬКО через `canon/index.js`** — не через отдельные файлы
2. **Изменение канона** требует обновления `CANON_VERSION` в `index.js`
3. **Confirmed** — стабильно, прошло верификацию
4. **Speculative** — обсуждается, не в коде

## 4. Сущности и константы

### Сущности

| Сущность | Описание | Жизненный цикл |
|---|---|---|
| **У.Е. (R.U.)** — Учётная Единица | Временный импульс признания. | 28 часов (24+4). Статусы: `active` → `burned` или `active` → `transferred`. |
| **У.М. (R.M.)** — Учётный Маркер | Неизменяемый «световой след». | Создаётся в момент `TRANSFER`. Не может сгореть. |
| **О.К.** — "Открытый Ключик" | Узел / участник. | Gestation → Active → Silence/Sleep. |

### Константы эмиссии

| Константа | Значение | Источник |
|---|---|---|
| `EMISSION_DAILY_MIN` | **3 У.Е.** | `emission-policy.js` |
| `EMISSION_DAILY_MAX` (участник) | **3–13 У.Е./день** (добровольный выбор) | `emission-policy.js` |
| `EMISSION_DAILY_MAX` (Глава ор𝕯ена ::02::–::09::)и глава от𝕯ела ::О0::-::О16:: | **29 У.Е.** (13 + 16) | `emission-policy.js` |
| `EMISSION_DAILY_MAX` (Предстоятель ор𝕯ена ::01::) | **до 32 У.Е.** (авто: 2 × 16 парт) | `emission-policy.js` |
| `SYSTEM_CAPACITY_26` | **26 У.Е.** — временная ёмкость системы в период наложения потоков (20:00–00:00 UTC). 13 max × 2 параллельных потока (active + impulse) = 26 максимальных У.Е. в воздухе одновременно. | Концептуальная константа |

### Метроном (фазы)

| Фаза | Время (UTC) | Описание |
|---|---|---|
| `ACTIVE_PHASE` | 04:00–19:55 | Нормальный режим. Эмиссия, трансферы, признание. |
| `SILENCE_PHASE` | **19:55–20:00** | Полная блокировка всех мутаций данных. |
| `SLEEP_PHASE` | **20:00–04:00** | У.Е. получают статус `impulse`, блокированы для передачи. |

### Прочие константы

| Константа | Значение | Источник |
|---|---|---|
| `BURN_WINDOW_ACTIVE` | 24h | `temporal.js` |
| `BURN_WINDOW_IMPULSE` | 28h | `temporal.js` |
| `SYSTEM_RESERVE` | ::00::–::33:: | `reserved.js` |
| `ORACLE_OK` | `::О::` | `reserved.js` |
| `TRIADS` | T1(1-3), T2(4-6), T3(7-9), T4(10-12), T5(21) | `ontology.js` |

## 6. Этические Стоп-краны

1. **Экзистенциальная ошибка:** Ущерб достоинству человека недопустим даже при логической эффективности кода.
2. **Пространство без счёта:** Отказ от цифровизации зон заботы, помощи и наставничества.
3. **Отказ от геймификации:** Запрет на рейтинги, соревнования и накопление У.Е. как капитала.
4. **Выход там же, где вход:** Участник может покинуть пространство в любой момент без обязательств и потерь.
