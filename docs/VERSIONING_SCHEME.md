# Версионирование — Pygmalion / C.R.I.S.T.A.L.L.

## Формат версии

```
epoch.phase.build.revision
```

| Компонент | Описание | Пример | Правила |
|-----------|----------|--------|---------|
| **epoch** | Эпоха / фундаментальная смена архитектуры | `0`, `1`, `2` | Инкремент только при ломании всего контракта |
| **phase** | Фаза внутри эпохи (Canon Layer) | `4`, `5`, `6` | 4 = Phase 1 Stable, 5 = Pilot 1000, 6 = Pilot 10000 |
| **build** | Номер сборки / итерация внутри фазы | `1000`, `2000` | Монотонно возрастает, сброс при смене phase |
| **revision** | Патч / хотфикс внутри сборки | `01`, `02` | Сброс при смене build |

### Примеры

| Версия | Смысл |
|--------|-------|
| `0.4.02` | Эпоха 0, Фаза 4 (Phase 1), Сборка 2, Ревизия 0 |
| `0.5.1000.01` | Эпоха 0, Фаза 5 (Pilot 1000), Сборка 1000, Ревизия 1 |
| `1.0.0.0` | Эпоха 1 (новая архитектура), Фаза 0, Сборка 0, Ревизия 0 |

---

## Правила сравнения компонентов

Сравнение **лексикографическое по компонентам** (как tuple of ints):

```
versionA > versionB  ⇔
  epochA  > epochB  ||
  epochA == epochB && phaseA  > phaseB  ||
  epochA == epochB && phaseA == phaseB && buildA  > buildB  ||
  epochA == epochB && phaseA == phaseB && buildA == buildB && revisionA > revisionB
```

- Компоненты парсятся как **целые числа** (leading zeros игнорируются: `01` = `1`).
- Отсутствующие компоненты считаются `0` (совместимость с `major.minor.patch`).

## Правила совместимости (Handshake)

PDA объявляет диапазон совместимости:

```json
{
  "compatibility": {
    "min": "0.5.1000.01",
    "max": "1.x"
  }
}
```

### Парсинг `min` / `max`

- `min` — **включительная** нижняя граница. Все версии `>= min` допустимы.
- `max` — **включительная** верхняя граница по компонентам до последнего явного.
  - `max: "1.x"` → разрешены все версии с `epoch == 1`, любой `phase/build/revision`.
  - `max: "1.0.x"` → разрешены `epoch==1, phase==0`, любой `build/revision`.
  - `max: "1.0.0"` → только точно `1.0.0.0`.

### Алгоритм проверки

```js
function inRange(version, min, max) {
  return cmp(version, min) >= 0 && cmp(version, max) <= 0;
}
```

Где `cmp` — компонентное сравнение с трактовкой `x` как "любое значение" (wildcard) для данного и всех младших компонентов.

## Правила задания диапазонов

| Шаблон | Означает |
|--------|----------|
| `1.x` | Любая версия эпохи 1 |
| `1.0.x` | Эпоха 1, фаза 0, любая сборка/ревизия |
| `1.0.5` | Эпоха 1, фаза 0, сборка 5, любая ревизия |
| `1.0.5.2` | Точная версия |

### Запрещено

- Использовать нестандартные префиксы (`v`, `phase1-` и т.д.) в `min`/`max`.
- Пропускать компоненты в `min` (должен быть полный 4-компонентный формат).
- Менять формат без синхронизации этого документа и `version-handshake.js`.
