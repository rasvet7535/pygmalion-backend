# CANON.md — Канон Пигмалион

## Терминология

| Термин          | Описание |
|-----------------|----------|
| **У.Е.**        | **Учётная Единица** — Recognition Unit (R.U.) — временный импульс признания (живёт 24–28 часов) |
| **У.М.**        | **Учётный Маркер** — Recognition Marker (R.M.) — неизменяемый след передачи, вечный элемент ro.DAG |
| **O.K.**        | **Открытый Ключик** — суверенный идентификатор участника (один человек — один О.К.) |
| **Triad**       | Группа R.U. (3 шт.), объединённых темой |
| **Gestation**   | Фаза активной эмиссии (04:00–19:55 UTC) |
| **Silence**     | Фаза тишины (19:55–20:00 UTC) — абсолютный блок всех актов |
| **Impulse**     | Фаза импульса (20:00–03:59 UTC) |
| **Predstoyatel**| Предстоятель — специальные ключи `::OP??EH ??AP::`, `::00::`, `::01::` |

**Важное правило:** Один «Открытый Ключик» = один человек. Это базовая политика платформы.

## Механика

Система работает в двухфазном режиме: **R.U. > R.M.**

1. Каждый Открытый Ключик получает право на ежедневную эмиссию **R.U.** (3–13 в день).
2. При передаче R.U. превращаются в неизменяемые **R.M.** (Recognition Markers).

## Фазы времени (Метроном 24+4)

- **Gestation (Active)**: 04:00–19:55 UTC — разрешены эмиссия и передача
- **Silence**: 19:55–20:00 UTC — абсолютный запрет всех актов
- **Impulse**: 20:00–03:59 UTC — разрешена только эмиссия, передача запрещена

У.Е., эмитированные в Impulse, живут до 28 часов (правило сна).

## Архитектурные принципы (SSOT)

- **acts_log** — единственный источник истины (append-only, immutable).
- **Preview > Execute** — все изменения только через канонические акты.
- **Replay** — полная восстановимость состояния из acts_log.
- **Silence** — защита от геймификации.
- **Burn** — 24ч (Active) / 28ч (Impulse).

**Единый SSOT** — `acts_log` + `canon/` (emission-policy.js, temporal.js). Все проекции (ue_units, um_markers, ro_dag_edges) восстанавливаются через Replay.
